## 設計上の重要な決定（Day59）

### WindowPlugin 統一パターン
```
【外側】MidiInputWrapper → TransportEvent { slot, value }（layerId なし）
【UI層】WindowPlugin → TransportEvent { slot, value, layerId, source:'window' }
```
- `source:'window'` のイベントは `layerId` を持つ → そのレイヤーのみに書く
- `source:'midi'` のイベントは `layerId` なし → 全レイヤーに書く
- Window が「自分は layer-N だ」と申告するだけ・振り分けは TransportManager が行う

### registrationKey 設計
- `'layer-N:geometry'` / `'layer-N:camera'` / `'layer-N:fx'` で独立管理
- 将来 PixiJS 等が追加されても `'layer-N:pixi'` で共存可能
- Registry はキーの意味を知らない
