## Day63 で完了したこと

### 1. cc-mapping.md v0.5（X桁新設・FX CC番号衝突解消）
- 番号体系を `[L][T][S][X][P]` の5桁構造に再定義
- X桁（プラグイン内識別番号）を新設し FX 12種の CC番号衝突を完全解消
- `cc-mapping-guide.md` v1.0 新規作成
- `pnpm gen:all` → 83 mappings 再生成

### 2. Standard Window 新規実装
```
src/plugins/windows/standard-window/
  RangeSlider.tsx              ← lo/hi カスタムdivドラッグ + 現在値スライダー
  GeometryStandardWindow.tsx
  CameraStandardWindow.tsx
  FxStandardWindow.tsx
  index.ts
```
- lo/hi はWindow側のlocalStateのみ（Registry/engine に影響しない）
- 送信変換: `t = (raw-min)/(max-min)` → `remapped = lo + t*(hi-lo)` → normalized
- APPLY後のパラメータ同期バグ修正（`useEffect` で `param.value` 追従）
- FxStandardWindow の `onFxChanged` unsubscribe バグ修正（タブ切り替え不具合）

### 3. WindowMode 型設計
- `src/types/windowMode.ts` 新規作成
- `geometry/camera/fx: 'none' | 'simple' | 'standard'`
- `macro: 'none' | 'macro-window'`
- `mixer: 'none' | 'mixer-simple'`
- `monitor: boolean`
- App.tsx の `uiVisible` を `windowMode` に一本化

### 4. PreferencesPanel 全面刷新
- レイアウト: PRESETS → WINDOWS → [L1/L2/L3] → SETUP → Macro/Mixer → Monitor
- WINDOWS: レイヤー非依存ドロップダウン（none/Simple/Standard）
- SETUP: 共有レイヤータブで Geometry/Camera/FX Plugin 選択
- パネルがドラッグ移動可能・縦スクロール対応
- Monitor を最下部のデバッグ専用チェックボックスに移動

### 5. キーバインド整理
- `1`: Macro トグル
- `3`: Mixer トグル
- `6`: Monitor トグル
- `2/4/5/7/8/9`: 廃止（Preferences で管理）

## 確定したアーキテクチャ（Day63 時点）

### Window 階層
```
Simple Window   → パラメータ表示・操作（min〜maxフル範囲）
Standard Window → パラメータ表示・操作 + lo/hi 稼働幅制限
（将来）D&D Window → MacroKnob アサイン用
```

### WindowMode 管理
```
App.tsx の windowMode state
  ↓ props
PreferencesPanel → ドロップダウンで切り替え
  ↓ onWindowModeChange
App.tsx → 対応する Window コンポーネントを条件レンダリング
```
