## 発生した問題と解決策

### 問題1: fileStore 移行で ClipGrid が空になる
- **原因**: ClipGrid が mount 時に `loadGrid()` を呼ぶが、fileStore がまだ未初期化
- **解決**: geoStore に `fileStoreReady` フラグを追加し、ClipGrid の useEffect が `fileStoreReady` を監視してから読み込む

### 問題2: 最初の fileStore 実装を revert した
- **原因**: `layerPresetStore.ts` の localStorage 版が正常動作していたのに、fileStore への移行実装が壊した
- **教訓**: 既存の動作実装を確認してから移行する。古い実装を再利用・流用すること

### 問題3: 「Mac のローカル保存」の言葉の混同
- **localStorage** = ブラウザ/Electron の内部 DB（Mac のファイルシステムではない）
- **Mac のローカル** = `~/Documents/GeoGraphy/` のファイルシステム
- Day82 の「localStorage 一本化」は言葉の混同による設計ミスだった

```

  geoAPI の役割 → プロジェクトファイル (Save/Open ダイアログ) のみ

  「📂 GeoGraphy フォルダを選択」ボタンが表示される
  ~/Documents/GeoGraphy/ を選択 → 以降は自動（IndexedDB にハンドル保存）

fileStoreReady フロー:
  App.tsx → initFileStore() → setFileStoreReady()
  ClipGrid → useGeoStore(s => s.fileStoreReady) を監視 → true になったら読み込み
```
