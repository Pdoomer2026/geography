## 確定したアーキテクチャ（Day62 時点）

### SDK 境界
```
【公開（Window が触れる）】
  engine.handleMidiCC()
  engine.getParameters() / getParametersLive()
  engine.onRegistryChanged()
  engine.getMacroKnobs() / setMacroKnob() 等
  useParam() / useAllParams() Hook

【非公開（engine 内部に隠蔽）】
  TransportRegistry / AssignRegistry / TransportManager
  ParameterStore / ccMapService / Clock
```

### GeoMonitor の設計原則
```
getParameters()     → Registry スナップショット（syncValue 経由）→ Window の購読用
getParametersLive() → plugin.params 直接読む（毎回最新）→ GeoMonitor 専用
```

### 型生成パイプライン
```
cc-mapping.md（SSoT）
  → pnpm gen:all
  → settings/cc-map.json（自動生成）
  → src/types/geo-types.generated.d.ts（型定義）
  → src/types/geo-cc-map.generated.ts（実行時定数）
```

## 発見した問題（次回対応）

### FX の CC 番号衝突（GeoMonitor が発見）
同一セマンティック（EXISTENCE/Primary Amount）の FX が全て CC14101 を共有しており、
同一レイヤー内で CC 衝突が発生している。

```
bloom     strength  CC14101  ← 衝突
zoom-blur strength  CC14101  ← 衝突
rgb-shift amount    CC14101  ← 衝突
film      intensity CC14101  ← 衝突
glitch    goWild    CC14101  ← 衝突
```

**対応方法:** `cc-mapping.md` の下2桁を FX ごとに一意化 → `pnpm gen:all` で解決
