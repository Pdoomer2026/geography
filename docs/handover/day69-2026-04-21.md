## アーキテクチャ方針（Day69 確定）

### Core 4（壁打ちで確定）
| ツール | 役割 |
|---|---|
| RxJS | Command 発火管理。スライダー操作・BPM tick を束ねて Application へ送る |
| Zustand | Registry の UI ミラー。engine を直接ポーリングしない |
| Zod | UI→Application 境界の実行時検証。外部 Plugin 対応時に拡張 |
| Tailwind | 表現のみ。catalog の `ui: "slider"` をどう見せるか |

### CommandStream フロー（確立済み）
```
スライダー操作 / Sequencer BPM tick
    ↓ paramCommand$.next()
commandStream.ts（RxJS Subject）
    ↓ throttleTime(16ms) ← App.tsx
engine.handleMidiCC()
```

### Zod が生きる境界
- UI → Application: `safeParse` で実行時検証（開発中バグ早期発見）
- 外部 JSON → Application: Plugin Store フェーズで本番化

### カタログ中心設計
```
コントリビューター
  config.ts に catalog を書く
      ↓ registry.register() 時に自動登録
paramCatalogRegistry.get('icosphere')
      ↓
UI / SDK がカタログを読んでスライダー自動生成（将来）
```

## 発生した問題と解決策

- **問題**: HANDOVER.md に「CLAUDE.md 更新」「src/core/CLAUDE.md 廃止注記」が次回タスクとして残っていた
  → **解決**: 実際には Day68 中に完了済みだった（ファイル確認で判明）

- **問題**: Radix UI が依存として残っていたが未使用
  → **解決**: 廃止して RxJS / Zustand に置き換え（Core 4 体制へ移行）

- **問題**: ModulatablePlugin に `catalog` を持たせたことで Engine に UI 情報が混入
  → **結論**: `ui / step / label` は Engine に不要だが、`min/max/default/requiresRebuild` は Engine 動作に必要。Plugin Store フェーズで「Engine metadata」と「UI metadata」に正式分離する暫定許容。
