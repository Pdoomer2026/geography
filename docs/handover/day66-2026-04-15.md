### 2. FxStack.applySetup() バグ修正（重要）
- **症状**: Preferences の APPLY を押した後、FX Window の Enable ボタンが描画に反映されなくなる
- **根本原因**: `applySetup()` で `enabledIds` 外の FX は `destroy()` のみで `create()` されず `pass = null` のまま。その後 `setEnabled(true)` しても `pass` が null なので `update()` 内で何もできなかった
- **修正**: 全 FX を `create()` して pass を維持し、`enabled` フラグだけで ON/OFF を制御するよう変更

```typescript
// 修正前: enabledIds の FX だけ create()
if (enabledSet.has(id)) {
  plugin.create(composer)
  plugin.enabled = true
} else {
  plugin.enabled = false  // pass = null のまま ❌
}

// 修正後: 全 FX を create() して pass を維持
plugin.create(composer)
plugin.enabled = enabledSet.has(id)  // ✅
```

### FxStack の ON/OFF 設計
- `setupFx()`: 全 FX を `create()` → pass 生成済み
- `applySetup()`: 全 FX を `destroy()` → 全 FX を `create()` → `enabled` で制御
- `setEnabled()`: `plugin.enabled` のみ変更（pass は維持）
- `update()`: 毎フレーム `pass.enabled = this.enabled` を設定
