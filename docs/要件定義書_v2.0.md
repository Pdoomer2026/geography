## 1. プロジェクト概要

### 1.1 コンセプト

GeoGraphy は「No-Texture・Plugin駆動・マルチライブラリ対応」の映像制作プラットフォームです。

単なる VJ ツールではなく、複数の描画ライブラリを Plugin として束ねるプラットフォームとして設計されています。Three.js・PixiJS・opentype.js など、異なるライブラリを Plugin Group として追加でき、ユーザーは必要な Plugin だけを有効化して自分専用の最小構成で起動できます。

アプリ名は「Geometry（幾何学）+ 地形（Geo）」と「Graph（グラフ・図形）+ 描写（Graphy）」のダブルミーニングです。

### 1.2 本質の再定義（Day33 確立）

```
表層:  Electron デスクトップ VJ ツール（ショーケース・サンドボックス）
本質:  No-Texture・Plugin駆動の Three.js ジオメトリ＆シェーダーアセットライブラリ
将来:  VR/AR アプリが GeoGraphy Plugin を import して使う
```

engine.ts が React 非依存であることで、VR/AR アプリでも Plugin をそのまま使い回せる。Command・Parameter Store → HMD コントローラーも同じ経路で流せる。

### 1.3 ビジュアルコンセプト

- レトロフューチャー・シンセウェーブ
- ワイヤーフレーム・グリッド・幾何学形状
- テクスチャを一切使わない・計算とベクターだけで完結
- 暗背景 + 発光するライン・形状・面

### 1.4 Goals

- Three.js によるリアルタイム 3D レンダリング
- Electron アプリとしてネイティブ配布（.dmg）
- Plugin Group 単位でライブラリを追加できるプラットフォーム設計
- CLI ツールで新規 Plugin を自動スキャフォールド
- MIDI 2.0 コントローラーによるリアルタイムパラメーター操作
- MacroKnob を中心とした統一パラメーター制御（MIDI / Sequencer / LFO の共通ハブ）
- CC番号 Rosetta Stone による全 Plugin 共通の操作体系
- Resolume Arena との連携を前提とした設計
- Program / Preview バス構造による本格的な映像ミキシング
- Shader Plugin（Graffiti コンセプト）による新しい表現
- Sequencer Plugin による BPM 同期パラメーター変調
- MixerPlugin による操作 UI の完全カスタマイズ（v2〜）
- Geometry Launcher・Sequencer による自動 VJ 運用（v2〜）
- Embed Export：映像を Web サイト・スクリーンセーバーに埋め込む（v2〜）
- OSS として公開・世界中のコントリビューターが Plugin を追加できる

### 1.5 非機能要件・対象外

**全バージョン対象外（変更不可原則）：**
- テクスチャ貼り付け（No-Texture 原則・全バージョン禁止）
- 動画再生（Resolume に任せる）
- クラウド同期・コラボレーション
- Tauri / ネイティブアプリ化（Electron で解決済み）
- スマートフォン最適化（v2 以降）

**将来検討（v4 以降）：**
- Claude.ai 自動トランジション
- Video Input Driver（Resolume 受信）

## 3. Electron アーキテクチャ（Day24〜27 確立）

### 3.1 採用理由

| 比較 | Web ホスティング | Electron |
|---|---|---|
| サーバー | 必要 | **不要** |
| 配布方法 | URL | **.dmg / .exe をダウンロード** |
| ファイル操作 | ブラウザ制限あり | **Node.js fs で自由** |
| プラグイン追加 | 困難 | **外部フォルダをスキャン可能** |
| メニューバー | なし | **ネイティブメニューバー** |

### 3.2 プロセス構成

```
electron/
  main.js       ← メインプロセス（ウィンドウ生成・ファイルアクセス・メニュー・MIDI 2.0 受信）
  preload.js    ← レンダラーと main の橋渡し（IPC・contextBridge）
src/            ← 既存 Vite + React コードそのまま
```

セキュリティ設定（必須）：
- `nodeIntegration: false`
- `contextIsolation: true`
- `sandbox: true`

### 3.3 Electron 固定部分（レンダラーが触れない）

| 処理 | 場所 | 理由 |
|---|---|---|
| メニューバーの定義（GeoGraphy / File / View） | main.js | Electron `Menu` API |
| ファイルの Save / Load / Save As... | main.js | Node.js fs 操作 |
| ネイティブダイアログ（open/save） | main.js | Electron `dialog` API |
| Recent ファイルの管理・永続化 | main.js | ファイルシステム管理 |
| 外部プロセス連携・ファイル書き出し | main.js | Node.js 領域 |
| **MIDI 2.0 受信** | **main.js** | **Node.js MIDI ライブラリ・IPC でレンダラーへ** |
| 録画ファイル保存（save-recording IPC） | main.js | Node.js fs 操作 |

### 3.4 レンダラー（React）に残すべき処理

| 処理 | 理由 |
|---|---|
| engine への操作（restoreProject / buildProject） | Three.js と密結合 |
| UI 状態管理 | React の責務 |
| Web MIDI API（MIDI 1.0 互換） | ブラウザ API |
| autosave の送受信ロジック（useAutosave.ts） | engine と密結合 |
| MacroKnob への値反映 | engine.parameterStore と密結合 |

### 3.5 メニューバー設計

| メニュー | 項目 |
|---|---|
| GeoGraphy | About / Preferences...（⌘,）/ Services / Hide / Quit |
| File | New（⌘N）/ Open...（⌘O）/ Open Recent > / Save（⌘S）/ Save As...（⌘⇧S）/ Close Window / Start Recording（⌘R）/ Stop Recording（⌘⇧R） |
| **View** | **全 Window の表示/非表示トグル（§5）** |

`electron/package.json` に `{ "type": "commonjs" }` を置く。ルートの `"type": "module"` と共存するための必須対応。

## 5. Window / Panel 設計（Day35 更新）

### 5.1 命名原則

| 名称 | 定義 | 例 |
|---|---|---|
| **Window** | Plugin エコシステムの UI・コントリビューターがデザインできる | Mixer Simple Window / FX Simple Window |
| **Panel** | アプリ固定の小窓・コントリビューターが触れない | Preferences Panel / **MacroKnob Panel** |
| **Simple Window** | 各 Plugin のデフォルト最小 UI・カスタム Window Plugin がないときのフォールバック | Mixer Simple Window |

### 5.2 Simple Window 一覧（v1）

| Simple Window 名 | 対応 Plugin | ファイル |
|---|---|---|
| Mixer Simple Window | MixerPlugin | `src/plugins/mixers/simple-mixer/MixerSimpleWindow.tsx` |
| FX Simple Window | FX Plugin | `src/ui/FxSimpleWindow.tsx` |

### 5.3 Panel 一覧（v1）

アプリ固定・コントリビューターが触れない。`src/ui/panels/` に配置。

| Panel 名 | 内容 | ファイル |
|---|---|---|
| Preferences Panel | Setup / Plugins / MIDI / Output 等 | `src/ui/panels/preferences/PreferencesPanel.tsx` |
| **MacroKnob Panel** | **MacroKnob 32個・MIDI 2.0 アサイン・MIDI Learn** | **`src/ui/panels/macro-knob/MacroKnobPanel.tsx`** |

各 Panel は固有の CLAUDE.md を持つ：
- `src/ui/panels/CLAUDE.md`（Panel 共通ルール）
- `src/ui/panels/preferences/CLAUDE.md`（Preferences 固有）
- `src/ui/panels/macro-knob/CLAUDE.md`（MacroKnob + MIDI 2.0 固有・最重要）

### 5.4 Window Plugin（v2〜）

各 Plugin に対してカスタム UI を差し込める仕組み。`src/plugins/windows/` に配置。コントリビューターが開発・PR で追加できる。有効化されると対応する Simple Window は非表示になる。

### 5.5 View メニュー

すべての Window・Panel は View メニューから表示/非表示を切り替えられる。

```
View
  ├── Mixer Simple Window     （⌘3）
  ├── FX Simple Window        （⌘2）
  ├── MacroKnob Panel         （⌘1）
  ├── ──────────────────────
  ├── Hide All Windows        （H）
  └── Show All Windows        （S）
```

## 8. プラットフォームアーキテクチャ

### 8.1 基本思想

GeoGraphy は「複数の描画ライブラリを Plugin として束ねるプラットフォーム」。エンジンは各 Plugin Group のライブラリを知らず、共通 Interface を通じてのみ通信する。

### 8.2 Plugin Group の概念

| Plugin Group | ライブラリ | バージョン | 主な用途 |
|---|---|---|---|
| Three.js Group | Three.js | v1 | 3D幾何学・ワイヤーフレーム・VJ映像 |
| PixiJS Group | PixiJS | v2 | 2D高速描画・フラットシェーディング |
| Typography Group | opentype.js | v3 | フォントアウトライン・Extrude・文字アニメーション |

### 8.3 PluginBase Interface（全 Plugin 共通）

```typescript
interface PluginBase {
  id: string
  name: string
  renderer: 'threejs' | 'pixijs' | 'opentype' | string
  enabled: boolean
}
```

### 8.4 Plugin 自動登録

```typescript
const modules = import.meta.glob('./*/index.ts', { eager: true });
Object.values(modules).forEach((m: any) => registry.register(m.default));
```

### 9.2 エンジン固定部分（Plugin が触れない）

| 固定部分 | 役割 |
|---|---|
| Parameter Store | 全パラメーターの一元管理・Command 経由のみ更新 |

## 11. MacroKnob / MIDI システム（Day35 大幅更新）

### 11.1 MacroKnob の位置づけ（コア固定）

MacroKnob は全 Plugin が依存する「値の集約点」。コントリビューターが触れない。Plugin 化しない。

```
【入力層】                     【MacroKnob層・コア固定】        【出力層】
MIDI 2.0 Controller  ──→
Sequencer Plugin     ──→  MacroKnobPanel（32個）   ──→  ParameterStore
LFO（v2〜）          ──→       core/macroKnob.ts            ↓
OSC（v2〜）          ──→                              Plugin.params
                                                   （Geometry/FX/Shader/Camera）
```

### 11.2 MIDI 2.0 対応

| 項目 | MIDI 1.0 | MIDI 2.0 |
|---|---|---|
| CC番号数 | 128個 | **32,768個** |
| 解像度 | 7bit | **32bit** |
| チャンネル数 | 16ch | **256ch** |
| 通信方向 | 一方向 | **双方向（MIDI-CI）** |
| デバイス認識 | 手動 | **Property Exchange（JSON自動）** |

**MIDI 2.0 受信は `electron/main.js` 経由（IPC）。** Web MIDI API は MIDI 1.0 のみ対応のため、Electron の Node.js レイヤーで処理する。

### 11.3 CC番号 Rosetta Stone（初期定義・確定後 `docs/spec/cc-standard.spec.md` に移管）

全 Plugin（Geometry / FX / Shader）がデフォルトで準拠する共通 CC 定義。シーケンサー「使い回し」と AI Orchestration の基盤。

| CC番号 | 抽象概念 | Geometry 例 | FX 例 | Shader 例 |
|---|---|---|---|---|
| CC 20 | Primary Amount | Size / Radius | Mix (Dry/Wet) | Emission |
| CC 21 | Density / Detail | Segments | Grain Size | Tiling |
| CC 22 | Deformation | Twist / Noise | Glitch | Displacement |
| CC 23 | Sharpness / Width | Stroke / InnerRadius | Contrast | Fresnel |
| CC 24 | Temporal Speed | Rotation Speed | Feedback Rate | UV Flow |

**CC定義は気長に・全 Plugin の params を比較しながら定義する。** 確定したら `docs/spec/cc-standard.spec.md` に書く（GeoGraphy の「憲法」）。

### 11.4 MIDI Learn

固定番号ではなく「デフォルト値として CC 20〜24 を割り当てる」設計。ユーザーが任意の番号に上書き可能。UI 上のパラメーターを右クリック → Learn → MIDI コンのノブを動かす → 自動アサイン。

### 11.5 MacroKnob Panel 仕様

| 項目 | 仕様 |
|---|---|
| ノブ数 | 32個（8ノブ × 4行）・アコーディオンで折りたたみ |
| 1ノブの最大割り当て | 3パラメーターまで |
| MIDI 範囲 | 0〜127（MIDI 1.0）/ 0〜4,294,967,295（MIDI 2.0） |
| 正規化 | 0.0〜1.0・Range（min/max）は各アサイン先で指定 |
| カーブ | v1：linear / v2：exp・log・s-curve |
| MIDI Learn | 右クリック → Learn モード |

## 12. Sequencer Plugin（Day35 新設・設計確定）

### 12.1 位置づけ

Sequencer は **Plugin**（コントリビューターが開発可能）。MacroKnob に値を送るだけで、Plugin.params を直接知らない疎結合設計。`src/plugins/sequencers/` に配置。

### 12.2 本質的な設計思想

> 「1小節のキャンバスに Shape（波形）を描く」

タイムラインに点を打つのではなく、**ループする Shape を描いて BPM に同期させる**。VJ 映像は反復が基本。参照モデル：Massive Stepper/Performer・ShaperBox 3。

### 12.3 アーキテクチャ

```
Sequencer Plugin
    ↓ macroKnobId に 0.0〜1.0 の値を送る
MacroKnob（コア固定）
    ↓ Range mapping（例: min=7, max=45）
ParameterStore
    ↓
```

## 13. Shader Plugin（Day34 新設・シーケンサー後実装）

### 13.1 コンセプト（Graffiti ビジョン）

Geometry は `material.opacity = 0`（透明）で位置データとして機能する。Shader Plugin は `engine.getGeometryData(layerId)` で頂点・エッジ・面を受け取り、その骨格をなぞることで「今まさに描いている」ように見える。

- 多角形のエッジを一筆書きでトレース → ストリートアートの「一発描き」
- 面をスプレーノイズで塗りつぶす → スプレー缶のリアルな質感

### 13.2 アーキテクチャ

```
Layer
├── GeometryPlugin（alpha=0 で透明・位置データ専用）
├── ShaderPlugin   （GeometryData を受け取り描画）
└── FX Stack       （ポストプロセッシング・スクリーン空間）
```

### 13.3 Effect Type

| 値 | 名前 | 内容 |
|---|---|---|
| 0 | Fill（面塗り） | ノイズ混じりの粒子で面を徐々に塗りつぶす |
| 1 | Outline（一筆書き） | エッジを `uProgress` に沿って光の粒子がなぞる |
| 2 | Detail（ディテール） | 液だれ・ハイライト・飛沫などの仕上げエフェクト |

### 13.4 実装タイミング

**シーケンサー完成後。** `uProgress（0.0〜1.0）` の制御主体（シーケンサー）が必要なため。
