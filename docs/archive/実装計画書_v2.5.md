## 1. 開発の基本方針

### 1.1 Claude Code との協業ルール

- 一度に全部作らない・1ファイルずつ確認しながら進める
- 必ず説明を受けてから実装する
- 動作確認を必ずはさむ
- 各モジュールの CLAUDE.md を読んでからコードを書く
- YouTube 動画で解説できる理解度を常に維持する

### 1.2 段階的な確認ポイント

| Day | 達成目標 | 確認内容 |
|---|---|---|
| Day 1 | ブラウザで黒い画面に回転する立方体が表示される | Three.js 動作確認 ✅ |
| Week 1 | grid-wave が固定カメラで表示される | Plugin System 動作確認 ✅ |
| Week 2 | MIDI ノブで波の形が変わる | 「これ VJ ソフトだ！」という瞬間 ✅ |
| Week 3 | Resolume に繋いで映像が出る | 「本物だ！」という瞬間 ✅ |

### 1.3 プラットフォーム思想（v2.5 追加）

- engine.ts は App.tsx に依存してはいけない・単体で動作できること
- Plugin には `renderer`・`enabled` フィールドを必ず持たせる
- Mixer Plugin は Window Plugin の一種・ただし閉じられない
- Transition Plugin は SceneState を受け取り変形するだけ・UI を持たない
- Program / Preview バスは v1 から本実装（差し込み口ではない）
- SimpleMixer は v1 で固定実装・ただし最初から MixerPlugin Interface に準拠する（v2 で Plugin 化するとき設計変更ゼロにするため）

## 4. Phase 2：Plugin / Driver アーキテクチャ

### 4.1 Interface 全定義（src/types/index.ts）

```typescript
// PluginBase（全 Plugin の共通基底）
interface PluginBase {
  id: string
  name: string
  renderer: 'threejs' | 'pixijs' | 'opentype' | string
  enabled: boolean
}

// Geometry / Particle / Light Plugin
interface GeometryPlugin extends PluginBase {
  create(scene: THREE.Scene): void
  update(delta: number, beat: number): void
  destroy(scene: THREE.Scene): void
  params: Record<string, PluginParam>
}
interface ParticlePlugin extends GeometryPlugin {}
interface LightPlugin extends GeometryPlugin {}

// FX Plugin
interface FXPlugin extends PluginBase {
  create(composer: EffectComposer): void
  update(delta: number, beat: number): void
  destroy(): void
  params: Record<string, PluginParam>
}

// Window Plugin
interface WindowPlugin extends PluginBase {
  component: React.FC
}

// SceneState（Program / Preview バスの核心）
interface SceneState {
  layers: LayerState[]
}
interface LayerState {
  geometryId: string
  geometryParams: Record<string, number>
  fxStack: FxState[]
  opacity: number
  blendMode: string
}

// Transition Plugin
interface TransitionPlugin extends PluginBase {
  duration: number
  category: 'pixel' | 'parameter' | 'bpm'
  execute(from: SceneState, to: SceneState, progress: number): void
  preview: string
}

// Mixer Plugin
interface MixerPlugin {
  id: string
  name: string
  renderer: string
  enabled: boolean
  component: React.FC  // 閉じられない Window Plugin
}

// Modulator Driver
interface ModulatorDriver {
  id: string
  name: string
  getValue(paramId: string): number | null
}
```

### 4.2 Command パターン（差し込み口）

```typescript
interface Command {
  execute(): void
  undo(): void
  description: string
}

class ParameterStore {
  set(key: string, value: number): void {
    this.history.execute(new SetParameterCommand(key, prev, value))
  }
  undo(): void { this.history.undo() }
  redo(): void { this.history.redo() }
}
```

## 6. Phase 4：FX + Light + Particle

### 6.1 FX スタック（順序厳守）

```
Three.js レンダリング
→ AfterImagePass → FeedbackPass → UnrealBloomPass
→ KaleidoscopePass → MirrorPass → ZoomBlurPass
→ RGBShiftPass → CRTPass → GlitchPass
→ ColorGradingPass（必ず最後）→ 出力
```

### 6.2 FX デフォルト設定

| FX | 起動時 | 初期値 |
|---|---|---|
| Bloom | ON | strength 0.8 / radius 0.4 / threshold 0.1 |
| After Image | ON | damp 0.85 |
| RGB Shift | ON | amount 0.001 |
| その他全 FX | OFF | --- |
| ColorGrading | ON | saturation 1.0 / contrast 1.0 / brightness 1.0 |

interface MacroAssign {
  paramId: string

  max: number
  curve: 'linear'

interface MacroKnobConfig {
  id: string
  name: string
  midiCC: number
  assigns: MacroAssign[]  // 最大3つ

const map = (midi: number, min: number, max: number) =>

## 12. v2 実装計画（v2.5 新設）

### 12.1 v2 実装順序

| 順序 | 内容 | 理由 |
|---|---|---|
| 1 | Geometry ランチャー（Window Plugin） | シンプル・すぐ現場で使える・効果大 |
| 2 | Program / Preview バス本格強化 | v1 の実装を Plugin 化に向けて整理 |

| 4 | CrossfadeMixer 実装 | Program / Preview 大画面・本格 VJ 向け |
| 5 | GL Transitions 移植（厳選10種） | Transition の選択肢を増やす |
| 6 | Embed Export 実装 | Web 展開・スクリーンセーバー対応 |
| 7 | PixiJS Plugin Group | 2D 塗り潰し図形・新しい表現 |
| 8 | Geometry シーケンサー（Window Plugin） | v2 の締めくくり・実験場 |

### 12.2 Geometry ランチャーの実装方針

SceneState のリストを順番に Preview バスに送り・クロスフェーダーを自動実行するだけ。

```typescript
const playlist: SceneState[] = [sceneA, sceneB, sceneC]
let current = 0

onBeat(4, () => {  // 4拍ごとに
  previewBus.setState(playlist[current])
  mixer.triggerCrossfade(selectedTransition)
  current = (current + 1) % playlist.length
})
```

## 13. 矛盾点・注意点（v2.5 整理）

| 懸念点 | 解決方針 |
|---|---|
| SimpleMixer は v1 で固定・v2 で Plugin 化する際に設計変更が必要 | v1 の時点から MixerPlugin Interface に準拠した実装にする |
| Program / Preview バスでメモリー懸念 | Preview は SceneState（JSON）+ 小キャンバス（320×180）のみ |
| 切り替え時に Three.js オブジェクトを生成するコストが発生 | 生成コストは実用上問題なし・FPS への影響を最初に確認する |
| GL Transitions の移植コスト | v2 では厳選10種のみ移植・コントリビューターに開放して増やす |
| Geometry ランチャーとシーケンサーの混同 | 別々の Window Plugin として実装 |
