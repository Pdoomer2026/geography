## 1. プロジェクト概要

### 1.1 コンセプト

GeoGraphy は Three.js を使用したブラウザベースのリアルタイム VJ（Video Jockey）プラットフォームです。幾何学パターン（Geometry）をリアルタイムで操作し、レトロフューチャーな 3D CG 映像を生成します。

アプリ名は「Geometry（幾何学）+ 地形（Geo）」と「Graph（グラフ・図形）+ 描写（Graphy）」のダブルミーニングです。

### 1.2 ビジュアルコンセプト

- レトロフューチャー・シンセウェーブ
- ワイヤーフレーム・グリッド・幾何学形状
- 精密な 3D モデルや Texture は使用しない
- Geometry の特性を最大限に活かした表現
- 暗背景 + 発光するライン・形状

### 1.3 Goals

- Three.js によるブラウザ上でのリアルタイム 3D レンダリング
- Geometry をプラグインとして管理・追加できる設計
- CLI ツールで新規 Geometry プラグインを自動スキャフォールド
- MIDI コントローラーによるリアルタイムパラメーター操作
- Resolume Arena との連携を前提とした設計
- OSS として公開・世界中のコントリビューターがプラグインを追加できる

### 1.4 非機能要件・対象外

全バージョン対象外：
- テキスト・タイポグラフィ機能（Three.js での日本語表示が困難・別アプリで開発予定）
- テクスチャ貼り付け（制作ソフトの領域・Blender / C4D 等で対応）
- 動画再生（Resolume に任せる）
- Syphon 連携（スコープ外）
- クラウド同期・コラボレーション
- スマートフォン最適化（v2 以降）

将来検討（v4 以降）：
- タイポグラフィアプリとの WebSocket 連携
- シンプルな背景テクスチャ（noise 等）

## 5. アーキテクチャ設計

### 5.1 基本思想

GeoGraphy のすべての機能は Plugin / Driver として設計されています。エンジンは各 Plugin / Driver の種類を知らず、Interface を通じてのみ通信します。これにより、コアを修正せずに機能を追加・拡張できます。

### 5.2 Plugin / Driver の種類

| 種類 | 役割 | Interface |
|---|---|---|
| Geometry Plugin | 何を描画するか | create / update / destroy / params |
| Particle Plugin | 背景・雰囲気を作るか | create / update / destroy / params |
| FX Plugin | どんなエフェクトをかけるか | create / update / destroy / params |
| Light Plugin | どんなライトを当てるか | create / update / destroy / params |
| Window Plugin | どんな UI を表示するか | React FC |
| Tempo Driver | どこからテンポを取るか | start / getTempo / stop |
| Input Driver | どのデバイスで操作するか | start / getInput / stop |
| Output Driver | どこに出力するか | start / render / stop |
| Modulator Driver | どこからパラメーター値が来るか | getValue / id / name |

### 5.3 Plugin 自動登録

Vite の import.meta.glob を使用し、plugins/ 以下のすべてのプラグインを起動時に自動登録します。

```typescript
// src/plugins/geometry/index.ts
const modules = import.meta.glob('./*/index.ts', { eager: true });
Object.values(modules).forEach((m: any) => registry.register(m.default));
```

### 5.4 Single Source of Truth

すべてのパラメーターは Parameter Store で一元管理されます。MIDI・GUI・AUTO のどこから変更しても同じ値が更新され、GUI ノブは常に現在値をリアルタイムで反映します。

### 5.5 レイヤーシステム（確定）

複数の Canvas を CSS position: absolute で重ねる方式を採用します。

| 項目 | 仕様 |
|---|---|
| 最大レイヤー数 | v1 は 3（config.ts の MAX_LAYERS で変更可能） |
| 合成方式 | CSS mixBlendMode（normal / add / multiply / screen / overlay） |
| 各レイヤー | 独立した Three.js インスタンス・独立した FX スタック |
| 操作切り替え | 下部パネルの [L1][L2][L3] ボタンで操作対象を切り替え |
| 出力 | localhost:5173?layer=1・?layer=all など URL パラメーターで選択 |

### 5.6 Command パターン（アンドゥ設計）

Parameter Store のすべての変更は Command 経由で行います。

```typescript
interface Command {
  execute(): void
  undo(): void
  description: string
}
```

| 設定 | 値 |
|---|---|
| 最大アンドゥ履歴 | 50ステップ（config.ts の MAX_UNDO_HISTORY で変更可能） |
| ショートカット | Cmd+Z（アンドゥ）/ Cmd+Shift+Z（リドゥ） |
| v1 対象 | パラメーター変更・Geometry 切り替え・Layer 追加削除・FX ON/OFF |
| 対象外（v1） | プリセット上書き（確認ダイアログで対応） |

## 8. Camera システム

### 8.1 カメラモード

| モード | 説明 | 実装方法 |
|---|---|---|
| A：オービット | 球面座標系で位置を計算 | PerspectiveCamera + 手動計算 |
| B：オブジェクト自転 | カメラ固定・オブジェクト回転 | Object3D.rotation |
| C：空撮ドローン | XZ 平面移動・Y 固定 | camera.position 直接操作 |
| D：フライスルー | Z 軸方向に前進 | camera.position.z -= speed |

### 8.2 AUTO モード

- 各カメラモードに重ねる ON/OFF オプション
- BPM に同期してカメラ位置をランダムに変化
- Lerp（線形補間）でスムースに移動
- 強度パラメーターで動きの大きさを制御

## 9. マクロノブ

### 9.1 概要

Massive のマクロコントロールを VJ 向けに最適化した設計です。1つのノブで複数のパラメーターを同時に操作できます。

### 9.2 仕様

| 項目 | 仕様 |
|---|---|
| ノブ数 | 32個（8ノブ × 4行）・アコーディオンで折りたたみ |
| 1ノブの最大割り当て | 3パラメーターまで |
| MIDI 範囲 | 0〜127（MIDI 標準） |
| 変化幅 | 各割り当てに min / max を個別指定 |
| カーブ | v1：linear / v2：exp・log・s-curve |
| 設定方法 | ノブ右クリック → Assign Parameter |
| 保存 | template-basic.md の Macro Knobs セクション |

## 15. Claude.ai インテグレーション

GeoGraphy のすべての設定は .md ファイルで管理されます。Claude.ai に話しかけるだけで設定を変更できます。

| ファイル | 自然言語の例 |
|---|---|
| template-basic.md | 「テクノっぽくして」「もっと幻想的に」 |
| preferences.md | 「M1 MacBook Air 用に軽量な設定にして」 |
| settings/mappings/midi/bcr2000.md | 「上段8つをマクロノブに割り当てて」 |
| settings/presets/[name].md | 「激しい・普通・落ち着いた の3プリセットを作って」 |
| settings/luts/ | 「映画的な色調に変えて」 |
