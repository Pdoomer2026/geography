## 1. 開発の基本方針

### 1.1 SDD × CDD 原則

- **SDD（Spec-Driven Development）**: 実装前に必ず `docs/spec/` の spec ファイルを読む。仕様変更はコードより先に spec を修正する
- **CDD（Compiler-Driven Development）**: 完了条件は `pnpm tsc --noEmit`（型エラーゼロ）+ `pnpm test --run`（全テストグリーン）の両方通過
- **`any` 禁止**: 型エラーは自律修正する
- **プラン先行**: 実装前にプランを提示・承認を得てから実装開始

### 1.2 開発サイクル

```
0. 【判定フェーズ】各モジュールの CLAUDE.md / docs/spec/[機能].spec.md を読む
   → 新規作成が必要か / 更新が必要か / 継続部分はそのまま触らない を判定
   → 必要な箇所だけ更新してから次へ進む
1. プランを提示・承認を得てから実装開始
2. 実装 → pnpm tsc --noEmit → 型エラーがあれば自律修正
3. pnpm test --run → 全グリーンになるまでループ
4. docs/progress/[task].log.md に完了を記録
5. 仕様変更が必要な場合 → spec を先に修正 → 再実装
6. セッション終了時に git commit + Day タグ（git tag dayN && git push origin dayN）
7. HANDOVER.md を更新
8. Obsidian dev-log に記録
```

## 4. 確立済みアーキテクチャ原則

### 4.1 engine.ts の制約（MUST）

- `engine.ts` は `App.tsx` に依存してはいけない・単体で動作できること
- Parameter Store の変更は必ず Command 経由のみ（直接代入禁止）
- Plugin には `renderer`・`enabled` フィールドを持たせること

### 4.2 Clock の2系統（MUST・混同禁止）

```typescript
engine.threeClock  // THREE.Clock → getDelta() でフレーム間経過時間
engine.clock       // Clock（独自）→ BPM・beat 値管理・readonly で外部公開
```

### 4.3 FX 実装ルール（MUST）

- `createFxPlugins()` ファクトリを使う（シングルトン共有禁止）
- `fxStack.update()` は `enabled=false` でもスキップしない（`pass.enabled` で制御）
- FX スタック順序は `FX_STACK_ORDER` 定数に従う

### 4.4 Electron アーキテクチャ（MUST）

- Save / Load / Recent / メニュー定義 → `main.js` に集約
- **MIDI 2.0 受信 → `main.js`（IPC）** ← Day35 追加
- engine 操作 / autosave → レンダラー（React）
- `isDev = !app.isPackaged`（`process.env.NODE_ENV` は使わない）

### 4.5 Window / Panel 命名原則（Day35 更新）

| 名称 | 定義 | 例 |
|---|---|---|
| **Window** | Plugin エコシステムの UI・コントリビューターがデザインできる | Mixer Simple Window / FX Simple Window |
| **Panel** | アプリ固定・コントリビューターが触れない・`src/ui/panels/` に配置 | Preferences Panel / MacroKnob Panel |
| **Simple Window** | 各 Plugin のデフォルト最小 UI・フォールバック | Mixer Simple Window |

### 4.6 MacroKnob（Day35 確立）

- **MacroKnob = コア固定**（Plugin 化しない・コントリビューターが触れない）
- 全入力源（MIDI / Sequencer / LFO）のルーター兼、値の正規化レイヤー
- MIDI 2.0 受信は `electron/main.js` 経由（IPC）・CC 32,768個・32bit 解像度
- Sequencer → MacroKnob ID に値を送る → ParameterStore（疎結合）

### 4.7 CC番号 Rosetta Stone（Day35 確立・初期定義）

全 Plugin がデフォルトで準拠する共通 CC 定義。確定後 `docs/spec/cc-standard.spec.md` に移管。

| CC番号 | 抽象概念 | Geometry 例 | FX 例 | Shader 例 |
|---|---|---|---|---|
| CC 20 | Primary Amount | Size / Radius | Mix (Dry/Wet) | Emission |
| CC 21 | Density / Detail | Segments | Grain Size | Tiling |
| CC 22 | Deformation | Twist / Noise | Glitch | Displacement |
| CC 23 | Sharpness / Width | Stroke / InnerRadius | Contrast | Fresnel |
| CC 24 | Temporal Speed | Rotation Speed | Feedback Rate | UV Flow |

> CC定義は気長に・全 Plugin の params を比較しながら定義する。

### 4.8 Transition Plugin（MUST）

- `execute()` は純粋関数として実装（副作用なし・戻り値は `SceneState`）
- CrossFade の category は `'parameter'`

### 4.9 WebGL / Canvas（MUST）

- `preserveDrawingBuffer: true` が必須（`drawImage` で WebGL canvas を読み取るため）
- canvas は `width/height` に `devicePixelRatio` を乗算・CSS サイズは変更しない（Retina 対応）

## 7. Phase 15：Sequencer Plugin 実装

### 7.1 位置づけ

Sequencer は **Plugin**（コントリビューターが開発可能）。MacroKnob に `0.0〜1.0` の値を送るだけで Plugin.params を直接知らない疎結合設計。

**実装タイミング：MacroKnob Panel 完成後。**

### 7.2 アーキテクチャ

```
Sequencer Plugin
    ↓ macroKnobId に 0.0〜1.0 の値を送る
MacroKnob（コア固定）
    ↓ Range mapping（例: min=7, max=45）
ParameterStore
    ↓
Plugin.params（Geometry / FX / Shader / Camera）
```

### 7.3 仕様

| 項目 | 内容 |
|---|---|
| ステップ数 | 1〜32（可変・タイムライン幅固定・仕切り線のみ変化） |
| ループ長 | 64拍固定（16小節×4拍）・Ableton Link 同期 |
| 波形タイプ v1 | hold / linear / sine / saw / saw-down / square |
| 波形タイプ v2 | random / custom（ベジェ編集） |
| 操作系 | ドラッグ＆ドロップ（デザインモード）+ ライブクリック（ライブモード） |
| 参照モデル | Massive Stepper/Performer・ShaperBox 3 |

## 11. ハマりポイント集（Day1〜35 の知見）

| 問題 | 原因 | 解決策 |
|---|---|---|
| FX ON/OFF が効かない | `fxStack.update()` で enabled=false をスキップ | `update()` は常に全 Plugin を呼ぶ・`pass.enabled` で制御 |
| 複数レイヤーで FX クラッシュ | `getAllFxPlugins()` シングルトンを共有 | `createFxPlugins()` ファクトリで毎回新インスタンス生成 |
| `IFxStack` 循環参照 | `types/index.ts` から `core/fxStack.ts` を import | `IFxStack` インターフェースを `types/index.ts` 内に定義 |
| App.tsx が engine を呼ばない | 旧 App.tsx が独自 Scene/Renderer を管理 | `engine.initialize(container)` → `engine.start()` のみに変更 |
| preload.js ES Module エラー | ルートに `"type": "module"` があるため | `electron/package.json` に `{ "type": "commonjs" }` を追加 |
| isDev 判定が誤る | `process.env.NODE_ENV` は Electron ビルドで信頼できない | `!app.isPackaged` を使う |
| `dist-electron/` が誤コミット | `.gitignore` に未追加 | `.gitignore` に追加・`git rm -r --cached` で追跡解除 |
| beat 値が急ジャンプ | `setTempo()` 時に `startTime` をリセットしていない | テンポ変更時に `startTime = performance.now()` でリセット |
| drawImage が黒くなる | `preserveDrawingBuffer` が false（デフォルト） | `THREE.WebGLRenderer` に `preserveDrawingBuffer: true` を設定 |
| canvas がぼやける | `width/height` に DPR を乗算していない | canvas サイズに `devicePixelRatio` を乗算・CSS サイズは変更しない |
| zsh で `#` をコマンドに含めるとエラー | zsh の history expansion | コメント行とコマンド行を分けて渡す |
