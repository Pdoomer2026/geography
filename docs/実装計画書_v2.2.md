## 1. 開発の基本方針

### 1.1 Claude Code との協業ルール

- 一度に全部作らない・1ファイルずつ確認しながら進める
- 必ず説明を受けてから実装する
- 動作確認を必ずはさむ
- 各モジュールの CLAUDE.md を読んでからコードを書く
- YouTube 動画で解説できる理解度を常に維持する

## 3. Phase 1：基盤構築

### 3.1 プロジェクトセットアップ

```bash
# React + TypeScript テンプレートで作成
pnpm create vite geography --template react-ts
cd geography
pnpm install

# Three.js
pnpm add three @types/three

# shadcn/ui のセットアップ（Tailwind CSS を含む）
pnpm add -D tailwindcss postcss autoprefixer
pnpm dlx shadcn-ui@latest init

# Framer Motion
pnpm add framer-motion

# 開発ツール
pnpm add -D vitest eslint prettier
```

### 3.2 React + Three.js の役割分担

| 担当 | 技術 | 内容 |
|---|---|---|
| Canvas エリア | Three.js | レンダリング・3D 描画・アニメーション |
| UI 全般 | React 18+ | メニューバー・パネル・フローティングウィンドウ |
| コンポーネント | shadcn/ui | ボタン・スライダー・ドロップダウンのベース |
| アニメーション | Framer Motion | ウィンドウ開閉・ノブの動き・トランジション |

```typescript
interface GeometryPlugin {

}
```

### 4.2 Command パターン（差し込み口）

```typescript
// src/core/command.ts
interface Command {
  execute(): void
  undo(): void
  description: string
}

// src/core/parameterStore.ts
class ParameterStore {
  set(key: string, value: number): void {
    // 必ず Command 経由で変更する
    this.history.execute(
      new SetParameterCommand(key, prev, value)
    )
  }
  undo(): void { this.history.undo() }
  redo(): void { this.history.redo() }
}
```

## 5. Phase 3：Camera システム

### 5.1 カメラモードの実装

| モード | 実装方法 | Three.js クラス |
|---|---|---|
| A：オービット | 球面座標系で位置を計算 | PerspectiveCamera + 手動計算 |
| B：オブジェクト自転 | カメラ固定・オブジェクト回転 | Object3D.rotation |
| C：空撮ドローン | XZ 平面移動・Y 固定 | camera.position 直接操作 |
| D：フライスルー | Z 軸方向に前進 | camera.position.z -= speed |

### 5.2 AUTO モード

- 各カメラモードに重ねる ON/OFF オプション
- BPM に同期してカメラ位置をランダムに変化
- Lerp（線形補間）でスムースに移動
- 強度パラメーターで動きの大きさを制御

### 5.3 Lerp 実装

```typescript
// カメラ位置の補間
camera.position.lerp(targetPosition, lerpFactor);
camera.quaternion.slerp(targetQuaternion, lerpFactor);
```

## 6. Phase 4：FX + Light + Particle

### 6.1 FX スタックの実装

```
Three.js レンダリング
→ AfterImagePass（残像）
→ FeedbackPass（フィードバック）
→ UnrealBloomPass（Bloom）
→ KaleidoscopePass（万華鏡）
→ MirrorPass（ミラー）
→ ZoomBlurPass（ズームブラー）
→ RGBShiftPass（RGB シフト）
→ CRTPass（CRT）
→ GlitchPass（グリッチ）
→ ColorGradingPass（色調整・必ず最後）
→ 出力
```

### 6.2 FX デフォルト設定（起動時）

| FX | 起動時 | 初期値 |
|---|---|---|
| Bloom | ON | strength 0.8 / radius 0.4 / threshold 0.1 |
| After Image | ON | damp 0.85 |
| RGB Shift | ON | amount 0.001（控えめ） |
| その他全 FX | OFF | --- |
| ColorGrading | ON | saturation 1.0 / contrast 1.0 / brightness 1.0（フラット） |

### 6.3 FX Plugin の各実装方針

| FX | 実装方法 |
|---|---|
| Bloom | three/examples/jsm/postprocessing/UnrealBloomPass |
| RGB Shift | ShaderPass + カスタム GLSL シェーダー |
| Glitch | three/examples/jsm/postprocessing/GlitchPass |
| Feedback | レンダーターゲットに出力を戻すループ |
| Kaleidoscope | ShaderPass + 角度で折り返す GLSL |
| CRT | ShaderPass + スキャンライン GLSL |
| After Image | three/examples/jsm/postprocessing/AfterImagePass |
| Zoom Blur | ShaderPass + 放射状ブラー GLSL |
| Mirror | ShaderPass + UV 座標反転 GLSL |
| ColorGrading | ShaderPass + Saturation / Contrast / Brightness GLSL |

## 8. Phase 6：Input Driver + Modulator Driver

### 8.1 各 Input Driver の実装

| Driver | API | 実装の核心 |
|---|---|---|
| MidiDriver | Web MIDI API | navigator.requestMIDIAccess() → CC メッセージをパラメーターにマッピング |
| TrackballDriver | Pointer Events API | pointermove でカメラ Pan / Tilt / Roll を制御 |
| GamepadDriver | Gamepad API | navigator.getGamepads() をポーリング・スティック値をカメラに反映 |

### 8.2 Modulator Driver の実装

| Driver | バージョン | 実装方法 |
|---|---|---|
| MidiModulator | v1 | MIDI CC 値を 0.0〜1.0 に正規化して返す |
| AutoModulator | v1 | BPM 同期・LFO 波形・Math.sin / Math.random |
| GuiModulator | v1 | ノブの GUI 値をそのまま返す（フォールバック） |
| AudioModulator | v2 | Web Audio API + FFT → Bass / Mid / Treble を返す |
| OscModulator | v3 | WebSocket 経由で OSC を受信 |

## 9. Phase 7：マクロノブ + レイヤーシステム

### 9.1 マクロノブの実装

```typescript
// src/ui/MacroKnob.tsx
interface MacroAssign {
  paramId: string
  min: number
  max: number
  curve: 'linear'  // v2 で exp / log / s-curve を追加
}

interface MacroKnobConfig {
  id: string
  name: string
  midiCC: number
  assigns: MacroAssign[]  // 最大3つ
}

// MIDI 0〜127 → 任意の範囲にマッピング
const map = (midi: number, min: number, max: number) =>
  min + (midi / 127) * (max - min)
```
