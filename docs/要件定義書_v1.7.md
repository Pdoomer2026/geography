## 1. プロジェクト概要

### 1.1 コンセプト（v1.5 改訂）

GeoGraphy は「No-Texture・Plugin駆動・マルチライブラリ対応」のブラウザベース映像制作プラットフォームです。

単なる VJ ツールではなく、複数の描画ライブラリを Plugin として束ねるプラットフォームとして設計されています。Three.js・PixiJS・opentype.js など、異なるライブラリを Plugin Group として追加でき、ユーザーは必要な Plugin だけを有効化して自分専用の最小構成で起動できます。

アプリ名は「Geometry（幾何学）+ 地形（Geo）」と「Graph（グラフ・図形）+ 描写（Graphy）」のダブルミーニングです。

### 1.2 ビジュアルコンセプト

- レトロフューチャー・シンセウェーブ
- ワイヤーフレーム・グリッド・幾何学形状
- テクスチャを一切使わない・計算とベクターだけで完結
- 暗背景 + 発光するライン・形状・面

### 1.3 Goals（v1.7 更新）

- Three.js によるブラウザ上でのリアルタイム 3D レンダリング
- Plugin Group 単位でライブラリを追加できるプラットフォーム設計
- Plugin の ON/OFF による最小構成起動
- CLI ツールで新規 Plugin を自動スキャフォールド
- MIDI コントローラーによるリアルタイムパラメーター操作
- Resolume Arena との連携を前提とした設計
- Program / Preview バス構造による本格的な映像ミキシング
- Mixer Plugin による操作 UI の完全カスタマイズ
- Transition Plugin による表現変形エンジン
- Geometry ランチャー・シーケンサーによる自動 VJ 運用
- Embed Export：映像を Web サイト・スクリーンセーバーに埋め込む
- OSS として公開・世界中のコントリビューターが Plugin を追加できる

### 1.4 非機能要件・対象外

全バージョン対象外：
- テクスチャ貼り付け（No-Texture 原則・全バージョン禁止）
- 動画再生（Resolume に任せる）
- クラウド同期・コラボレーション
- Tauri / ネイティブアプリ化（ブラウザで全て実現可能）
- .saver 形式ビルド（WebviewScreenSaver で解決済み）
- Syphon 連携（Browser Bridge ユーティリティで解決）
- スマートフォン最適化（v2 以降）

将来検討（v4 以降）：
- Claude.ai 自動トランジション
- Video Input Driver（Resolume 受信）

## 5. プラットフォームアーキテクチャ（v1.5 新設）

### 5.1 基本思想

GeoGraphy は「複数の描画ライブラリを Plugin として束ねるプラットフォーム」です。エンジンは各 Plugin Group のライブラリを知らず、共通 Interface を通じてのみ通信します。

### 5.2 Plugin Group の概念

ライブラリ単位で Plugin Group を構成します。

| Plugin Group | ライブラリ | バージョン | 主な用途 |
|---|---|---|---|
| Three.js Group | Three.js | v1 | 3D幾何学・ワイヤーフレーム・VJ映像 |
| PixiJS Group | PixiJS | v2 | 2D高速描画・フラットシェーディング |
| Typography Group | opentype.js | v3 | フォントアウトライン・Extrude・文字アニメーション |

### 5.3 Plugin ON/OFF・最小構成起動

ユーザーは必要な Plugin だけを有効化して起動できます。

| 用途 | 有効にする Plugin |
|---|---|
| VJ・幾何学映像 | Geometry / Particle / FX / Light |
| モーショングラフィックス（v2） | 2D Shape / FX |
| タイポグラフィ映像（v3） | Typography / FX |
| フル構成 | 全 Plugin 有効 |

### 5.4 Plugin Interface（共通 PluginBase）

全 Plugin は PluginBase を継承します。

| フィールド | 型 | 説明 |
|---|---|---|
| id | string | Plugin の一意識別子 |
| name | string | 表示名 |
| renderer | 'threejs' \| 'pixijs' \| 'opentype' \| string | 使用するライブラリを識別（v1.5 追加） |
| enabled | boolean | Plugin の有効・無効（v1.5 追加） |

## 6. Plugin / Driver 設計

### 6.1 基本思想

すべての機能は Plugin / Driver として設計されています。エンジンは各 Plugin / Driver の種類を知らず、Interface を通じてのみ通信します。

### 6.2 Plugin / Driver の全種類（v1.7 確定版）

| 種類 | 役割 | UI | v1 |
|---|---|---|---|
| Geometry Plugin | 何を描画するか（3D幾何学） | Window Plugin で操作 | ✅ |
| Particle Plugin | 背景・雰囲気（パーティクル） | Window Plugin で操作 | ✅ |
| FX Plugin | エフェクト（Bloom・グリッチ等） | Window Plugin で操作 | ✅ |
| Light Plugin | ライティング（環境光・ポイントライト等） | Window Plugin で操作 | ✅ |
| 2D Shape Plugin | 2D 塗り潰し図形（PixiJS） | Window Plugin で操作 | v2 |
| Typography Plugin | フォント・Extrude（opentype.js） | Window Plugin で操作 | v3 |
| **Mixer Plugin** | Program/Preview の操作 UI・フェーダー・クロスフェーダー | それ自体が UI | v1固定→v2Plugin化 |
| **Transition Plugin** | 映像・パラメーターの変形トランジション（Mixer のプルダウンで選択） | UI なし・処理のみ | v1最小実装 |
| **Window Plugin** | Geometry・FX・Particle 等の専用コントローラー UI | それ自体が UI | ✅ |
| Tempo Driver | テンポ入力（Ableton Link・タップ・手動） | なし | ✅ |
| Input Driver | デバイス入力（MIDI・トラックボール・ゲームパッド） | なし | ✅ |
| Output Driver | 出力先（HDMI・Resolume・NDI・Spatial） | なし | ✅ |
| Modulator Driver | パラメーター値の入力源（LFO・エンベロープ等） | なし | ✅ |
| Video Input Driver | 外部映像をレイヤーとして取り込む | なし | v4 |

> ※ v1 は SimpleMixer（固定実装）+ Beat Cut / CrossFade（最小 Transition）のみ。Mixer Plugin の交換・追加は v2 から。

### 6.3 エンジン固定部分（Plugin が触れない）

| 固定部分 | 役割 |
|---|---|
| Parameter Store | 全パラメーターの一元管理 |
| Plugin Registry | Plugin の登録・切り替え・list() |
| Command パターン | アンドゥ・リドゥ |
| レンダリングループ | requestAnimationFrame・delta・beat の供給 |
| BPM クロック | Tempo Driver からの BPM・beat 値の管理 |
| **メニューバー** | File / View / Plugins / Help（全 Mixer 共通・固定） |
| **マクロノブパネル** | 32ノブ・4列（MIDI との物理対応関係があるため固定） |

### 6.4 Plugin 自動登録

```typescript
// src/plugins/geometry/index.ts
const modules = import.meta.glob('./*/index.ts', { eager: true });
Object.values(modules).forEach((m: any) => registry.register(m.default));
```

### 6.5 Single Source of Truth

すべてのパラメーターは Parameter Store で一元管理されます。MIDI・GUI・AUTO のどこから変更しても同じ値が更新されます。

## 7. Program / Preview バス構造（v1 実装）

### 7.1 概要

v1 から Program バスと Preview バスの2系統を実装します。

| バス | 役割 | 実体 |
|---|---|---|
| Program バス | 今出力中の映像・Resolume 等に送られる | フルサイズ Three.js Scene |
| Preview バス | 次に出す映像・エディット中 | SceneState（JSON）+ 小サムネイルキャンバス（320×180） |

### 7.2 Preview バスのメモリー設計

- Preview バスは SceneState（パラメーターのみの JSON）として保持
- Three.js オブジェクトは Program に昇格した瞬間に生成・切り替え後に旧 Program を dispose()
- GPU メモリーの増加はほぼゼロ
- サムネイル確認用に小さなキャンバス（320×180）を1つだけ持つ

### 7.3 モニタリングモード

| モード | 用途 | 表示 |
|---|---|---|
| 通常モード | VJ スタンダード | Program 大・Preview 小（サムネイル） |
| 準備モード | 次の映像を大きく確認したい | Program 小・Preview 大 |
| デュアルモニター | 本格 VJ 運用 | モニター1：Program フル / モニター2：Preview フル |

### 7.4 切り替えフロー

```
Preview バスで次の映像を準備・エディット
  ↓
Mixer Plugin でトランジションを選択（プルダウン）
  ↓
クロスフェーダーで切り替え実行（MIDI フェーダーまたはマウス）
  ↓
Transition Plugin が SceneState を変形しながら移行
  ↓
Preview が Program に昇格・旧 Program が Preview に降格
```

### 7.5 SceneState Interface（核心）

```typescript
interface SceneState {
  layers: LayerState[]
}

interface LayerState {
  geometryId: string
  geometryParams: Record<string, number>
  fxStack: FxState[]
  opacity: number
  blendMode: string
}
```

## 9. Mixer Plugin（v2 実装）

### 9.1 公式 Mixer Plugin

| Plugin 名 | コンセプト | バージョン |
|---|---|---|
| SimpleMixer | 縦フェーダー × レイヤー数・最小構成・デフォルト | v1（固定）→ v2（Plugin 化） |
| CrossfadeMixer | Program / Preview 大画面 + クロスフェーダー・本格 VJ 向け | v2 |
| （コントリビューター製） | DJ ミキサー風・スイッチャー風・何でもあり | v2以降 |

### 9.2 Mixer Plugin の制約

- **閉じることができない**（常時表示・全 Mixer Plugin 共通）
- Transition Plugin の選択 UI（プルダウン）を必ず持つ
- クロスフェーダーを必ず持つ
- エンジン API を通じてのみ Parameter Store を操作する
- v1 の時点から MixerPlugin Interface に準拠した実装にすること（v2 で設計変更ゼロにするため）

### 9.3 MixerPlugin Interface

```typescript
interface MixerPlugin {
  id: string
  name: string
  renderer: string
  enabled: boolean
  component: React.FC  // 閉じられない Window Plugin
}
```

### 17.2 Geometry シーケンサー（v2 開発の中心）

- パラメーターを時系列で並べて BPM に変調する実験的な機能
- ステップ・カーブでパラメーターの時系列変化を定義
- Tempo Driver・Modulator Driver と深く統合
- v2 の締めくくり・コントリビューターの実験場

## 18. Embed Export（v2 実装）

- geography-embed.min.js（使用 Plugin のみ含む最小バンドル）
- Script タグ・iframe・URL の3形式出力
