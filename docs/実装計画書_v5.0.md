## 1. 開発の基本方針

### 1.1 SDD × CDD 原則

- **SDD**: 実装前に必ず `docs/spec/` の spec ファイルを読む。仕様変更はコードより先に spec を修正する
- **CDD**: 完了条件は `pnpm tsc --noEmit`（型エラーゼロ）+ `pnpm test --run`（全テストグリーン）の両方通過
- **`any` 禁止**: 型エラーは自律修正する
- **プラン先行**: 実装前にプランを提示・承認を得てから実装開始
- **コード確認優先**: ドキュメントの記述とコードが乖離している場合はコードを正とする

### 1.2 開発サイクル

```
1. docs/spec/[機能].spec.md を確認（または新規作成）
2. プランを提示・承認を得てから実装開始
3. 実装 → pnpm tsc --noEmit → 型エラーがあれば自律修正
4. pnpm test --run → 全グリーンになるまでループ
5. docs/progress/[task].log.md に完了を記録
6. セッション終了時に git commit + Day タグ（git tag dayN && git push origin dayN）
7. HANDOVER.md を更新
8. Obsidian dev-log に記録
```

## 4. 確立済みアーキテクチャ実装原則

### 4.1 engine.ts の制約（MUST）

- `engine.ts` は `App.tsx` に依存してはいけない・単体で動作できること
- Parameter Store の変更は必ず Command 経由のみ（直接代入禁止）
- Plugin には `renderer`・`enabled` フィールドを持たせること

### 4.2 FX Plugin 実装ルール（MUST・コード確認済み）

- 全 FX の `enabled` は **コード上デフォルト `false`**（Day79 コード確認済み）
- `createFxPlugins()` ファクトリを使う（シングルトン共有禁止）
- `fxStack.update()` は `enabled=false` でもスキップしない（`pass.enabled` で制御）
- `applySetup()` は全 FX を `create()` してから `enabled` フラグで制御する（Day66 確立）
- FX スタック順序（`FX_STACK_ORDER`）:
  AfterImage → Feedback → Bloom → Kaleidoscope → Mirror → ZoomBlur → RGBShift → CRT → Glitch → Film → FreiChen → ColorGrading（必ず最後）

### 4.3 Transport Architecture（Day58〜76 確立）

```
【外側】MidiInputWrapper（MIDI → TransportEvent）
【境界】TransportEvent { slot, value, source?, time? }
【コア】TransportManager → ParameterStore（GeoParamAddress をキー） → engine
【UI層】Window → paramCommand$.next() → throttleTime(16ms) → engine.handleMidiCC()
```

slot エンコーディング（Day76 確定）:
- CC 空間: `slot = channel * 128 + cc`（0〜2047）
- Note 空間: `slot = 2048 + channel * 128 + note`（2048〜4095）

### 4.4 GeoParamAddress（Day73〜74 確立）

```
フォーマット: geo://{layerId}/{pluginId}/{paramId}
例: geo://layer-1/icosphere/scale
ParameterStore のキー・MacroAssign の target・AI 向け一意識別子・OSC 互換
```

### 4.5 Camera Plugin 設計原則（Day44〜45 確立）

- Camera は独立した Plugin（Geometry に埋め込まない）
- レイヤーごとに独立したインスタンス（`getCameraPlugin()` がファクトリ呼び出し）
- `isCameraUserOverridden` フラグで自動連動とユーザー意図を両立

### 4.6 requiresRebuild フラグ（Day46 確立）

- メッシュ形状が変わる param には `requiresRebuild: true` を必ず設定
- `engine.setGeometryParam()` が検知して `layerManager.rebuildPlugin()` を呼ぶ（destroy → create）

### 4.7 Clock の2系統（MUST・混同禁止）

```typescript
engine.threeClock  // THREE.Clock → getDelta() でフレーム間経過時間
engine.clock       // Tone.js ベース IClock → BPM・beat・累積拍数管理
                   // getTotalBeats() が Sequencer の唯一の制御源
```

### 4.8 Core 4 体制（Day69 確立）

| ツール | 役割 |
|---|---|
| RxJS | `paramCommand$`：スライダー操作を throttleTime(16ms) で engine に送る |

| Zod | UI→Application 境界の実行時検証（DragPayload / AssignProposal / RangeConstraint 等） |
| Tailwind | 表現のみ |

### 4.9 Electron 薄い鏡アーキテクチャ（Day77〜78 確立）

```
【鏡 1】React UI メニュー（開発・Vercel デモ用）
  ボタンクリック → Application 層（projectManager）に直結

【鏡 2】Electron ネイティブメニュー（製品・.dmg 用）
  Menu click → sendToRenderer() のみ
  → App.tsx が geoAPI 経由で Application 層を呼ぶ

```

## 9. ハマりポイント集（Day1〜79 の知見）

| 問題 | 原因 | 解決策 |
|---|---|---|
| FX ON/OFF が効かない | `fxStack.update()` で enabled=false をスキップ | `update()` は常に全 Plugin を呼ぶ・`pass.enabled` で制御 |
| 複数レイヤーで FX クラッシュ | `getAllFxPlugins()` シングルトンを共有 | `createFxPlugins()` ファクトリで毎回新インスタンス生成 |
| `IFxStack` 循環参照 | `types/index.ts` から `core/fxStack.ts` を import | `IFxStack` インターフェースを `types/index.ts` 内に定義 |
| preload.js が ES Module エラー | ルートに `"type": "module"` があるため | `electron/package.json` に `{ "type": "commonjs" }` を追加 |
| isDev 判定が誤る | `process.env.NODE_ENV` は Electron ビルドで信頼できない | `!app.isPackaged` を使う |
| beat 値が急ジャンプ | `setTempo()` 時に `startTime` をリセットしていない | Tone.js 移行（Day73）で解消 |
| drawImage で WebGL canvas が真っ黒 | WebGL のデフォルトは描画後バッファをクリア | `THREE.WebGLRenderer` に `preserveDrawingBuffer: true` を追加 |
| edit_file の oldText がマッチしない | macOS APFS が日本語を NFD で保存するが Claude は NFC で送る | `python3 /Users/shinbigan/nfc_normalize.py` で NFC 正規化 |
| Camera Plugin が複数レイヤーで状態干渉 | ファクトリを呼ばずシングルトンを共有 | `getCameraPlugin()` がファクトリ呼び出しで毎回独立インスタンス |
| Geometry param 変更で見た目が変わらない | segments 等はメッシュ再構築が必要 | `requiresRebuild: true` フラグ + `rebuildPlugin()` で destroy→create |
| L2 スライダーが L3 に影響 | TransportEvent に layerId がなかった | `layerId?` 追加・source:'window' は該当レイヤーのみに書く |
| fxStack.applySetup() 後 FX が再 ON できない | enabledIds 外の FX は `pass = null` のまま | 全 FX を `create()` してから `enabled` フラグで ON/OFF（Day66） |
| onRegistryChanged × onPluginApply 無限ループ | 副作用の前に ref を更新していなかった | `pluginIdRef.current = geo.id` を副作用呼び出しの前に書く（Day74） |
| loHiMapRef のキーが衝突 | `param.ccNumber` だけをキーにしていた | `pluginId:ccNumber` の複合キーに変更（Day75） |
| FX Plugin のコードコメントと実態が乖離 | コメントに `enabled=true` と書いてあるが実際は `false` | コードを正とする。コメントは要修正（Day79 確認） |
| zsh で `-m` の日本語複数行がフリーズ | `dquote>` に入る | `.claude/dayN-commit.txt` に書いて `git commit -F` で実行 |
| zsh でインラインコメント（#）がエラー | zsh のインタラクティブモードはコメント非対応 | コメント行とコマンド行を分けて渡す |
