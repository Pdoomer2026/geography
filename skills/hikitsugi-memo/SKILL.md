# 引き継ぎメモスキル（GeoGraphy 開発セッション終了）

## 目的

毎回の開発セッション終了時に：
1. 今日の実装内容を HANDOVER.md の【継続層】に追記する
2. 次回のタスクを【引き継ぎ層】に上書き保存する
3. 次回の始業時に即座に再開できる状態にする

---

## 手順

### Step 1: 今日の実装内容を整理する

会話履歴から以下を抽出する：
- 実装したファイル一覧（新規 / 変更）
- 最後のコミットハッシュとメッセージ
- テスト結果（グリーン/レッド・テスト数）
- ブラウザ動作確認の状態
- 未解決の問題

### Step 2: HANDOVER.md の現在の内容を読む

filesystem ツールで `/Users/shinbigan/geography/HANDOVER.md` を読み込む。

確認する情報：
- 継続層の「実装済み内容」テーブルの最後の行（Day番号を確認）
- 引き継ぎ層の内容（上書き対象）

### Step 3: 引き継ぎメモ案をチャットに表示する

以下の構造で引き継ぎメモ案を作成し、チャットに表示する：

#### 継続層への追記（実装済みテーブルに1行追加）
```
| Day[N] | [実装内容の要約] | ✅ |
```

#### 引き継ぎ層（全体を上書き）
```markdown
### 🔴 次のセッションで最初にやること

1. `pnpm test --run` でグリーン確認（[テスト数] tests）
2. `pnpm dev` でブラウザ動作確認（[確認項目]）
3. Day [N+1] の実装タスクに進む

### 現在の作業状態

- **ブランチ**: `[ブランチ名]`
- **最後のコミット**: `[コミットメッセージ]`（[ハッシュ]）
- **動作確認状態**: [確認済み/未確認]
- **未コミットファイル**: [あり/なし]
- **開発環境**: Cursor / Claude Code（ターミナルで `claude` コマンドで起動）

### 未解決の問題

[なし / 問題の説明]

### 次回の本実装タスク（Day [N+1]）

1. **[タスク名]**
   - [具体的なアクション]
   - [具体的なアクション]

2. **[タスク名]**
   - [具体的なアクション]

3. `git add -A && git commit -m "feat: Day[N+1] - [内容]"`

### 今回のセッションで確定したこと

- [設計決定・ルール変更・新たな発見]
- [設計決定・ルール変更・新たな発見]
```

### Step 4: ユーザーに確認を求める

チャットに以下を表示する：

> 上記の内容で HANDOVER.md を更新してよいですか？

ユーザーから「はい」「OK」「お願いします」などの承認を受ける。

### Step 5: HANDOVER.md に書き込む

ユーザーの承認後、filesystem の `write_file` ツールで HANDOVER.md 全体を書き込む。

**注意**: `str_replace` ではなく `write_file` で全体を書き直すこと（パス解決の問題を避けるため）。

### Step 6: 完了メッセージを表示する

```
✅ HANDOVER.md を更新しました。

Day [N] お疲れ様でした！
次回は「始業」と声かけするだけで DAY[N+1].md が自動生成されます。
```

---

## 注意事項

- 【継続層】は追記のみ・過去の内容を消してはいけない
- 【引き継ぎ層】は毎回上書き・前回の内容は消してよい
- 書き込み前に必ずユーザーに内容を確認してもらう（HANDOVER_TEMPLATE.md のルール）
- 「今回のセッションで確定したこと」には設計変更・ルール追加・新たな発見のみ書く
- タスクは「NG：デバッグを続ける」「OK：registry.ts の register() を修正する」のようにアクションレベルで書く
- `write_file` でファイル全体を書き直す（継続層の過去内容は必ず保持する）
